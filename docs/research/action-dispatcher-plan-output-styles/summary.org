* Findings

** Immediate Codex also added =call_action=

The earlier analysis treated =call_action= in =hemingway-v3= as scope drift.
That was wrong.
The immediate Codex artifact at transcript idx 552 also added =call_action=.

The immediate Codex plan says:

- the goal covers calling actions from Elisp and from a model-facing tool;
- the design adds a model-facing bridge;
- the bridge is the =call_action= tool;
- acceptance criteria require =call_action= dispatch and compact-session refresh metadata.

The final committed Codex plan removed that scope only after the user corrected Codex:

#+begin_quote
Is it a tool? We don't need more tools, we already have elisp tool. It should just make the elisp interface nices for the agents to use
#+end_quote

So the fair replay control is =codex-immediate-after-replay-prompt/plan.org=, not =codex-original/plan.org=.

** Replays kept the same broad feature idea but expanded it

All replayed styles followed the same broad direction as immediate Codex: action descriptors, =e-actions-call=, standard action conversion, guidance updates, docs, tests, and the =call_action= bridge.

The main difference is size and structure.
Immediate Codex is 389 words.
The smallest replay is =caveman= at 1,607 words.
=hemingway-v3= is 1,857 words.

The replayed plans read like pre-implementation implementation plans.
Immediate Codex reads like a compact feature ledger artifact.

** Hemingway v3 is the best default among the E styles

=hemingway-v3= is not the closest artifact to immediate Codex by length.
=caveman= is shorter.
But =hemingway-v3= gives the best balance for general agent work:

- clear prose;
- low padding;
- enough project terms to stay precise;
- less verbosity than =hemingway-current= and =hemingway-suggested-v2=;
- better readability than =caveman= for multi-step implementation plans.

It produced a full plan rather than a compact ledger entry.
That is a prompt/control issue, not mainly a prose-style failure.
If we need Codex-like compact feature artifacts, the prompt should ask for a compact post-hoc ledger shape.

** Caveman compresses words but increases fragmentation

=caveman= has the fewest words among replayed outputs.
It has the most lines and bullets.
It compresses sentence content but expands visual structure.
This makes it useful as a stress test, not as the default style.

** Final Codex differs because of user correction, not initial model instinct

The final =codex-original= artifact is not the fair control for replay generation at the cut point.
It reflects a later human product decision: no generic model-facing action tool.

That correction is important for the feature.
It should not be counted as evidence that replay styles uniquely drifted into =call_action=.
Immediate Codex drifted there too.

* Recommendation

Adopt =hemingway-v3= as the default =hemingway= style.
Keep =hemingway-v3= as a named style for explicit comparisons and backward traceability.
Keep =hemingway-suggested-v2= as a verbose planning style.
Keep =caveman= as a compression stress test.

For future feature-plan replay studies, compare against both controls when available:

1. immediate Codex artifact at the same prompt point;
2. final corrected artifact after human steering.

Do not judge replay scope drift against final corrected artifacts unless the replay prompt includes the same human correction.

* Notes on validity

This is an approximate replay.
It cannot reproduce Codex's hidden context or exact model state.
It does preserve the visible request, diagnostic context, implementation trace before artifact creation, repository state, and output-style injection.
Each run verified that the intended style guidance was active before prompting.

The pre-action-dispatch commit =839e9c8= has no =e-actions-call=, no =e-action-create=, and no =call_action= matches in =lisp=, =docs=, or =test=.
That commit is the correct implementation baseline for reasoning about what the model had to design.
