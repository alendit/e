* Guidance Verification

The first comparison was invalid. I had tried to set =output-style= through harness-local runtime config. The output-style capability resolves its instructions inside the layer factory from global or directory capability config, so every first-run variant inherited the global =hemingway= style. Those invalid artifacts are archived under =invalid-runtime-override-20260628T211616Z/=.

The current runs were serialized. Before each harness and session were created, =e-capability-config= was set to that variant and kept set until =e-harness-prompt= returned. Each run wrote =effective-guidance.txt= and stores the same preflight in =metrics.json=. The preflight records the effective =output-style= capability instructions and whether they matched the expected guidance. All four current runs have =matches-expected: true=.

* Notes
