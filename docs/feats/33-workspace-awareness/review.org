* Review loop

The implementation adds a presentation-owned workspace service, chat and Org
canvas workspace affinity, workspace-aware display routing for first-party
shells, model-facing workspace actions, and a readable
=e://workspace-awareness/skills/workspace-display= guidance resource.

The first subagent review found four actionable issues:

- =e-workspace--display-action= returned =(FUNCTIONS ALIST)= instead of the
  =display-buffer= action shape =(FUNCTIONS . ALIST)=, so action alist entries
  such as =reusable-frames= and =inhibit-switch-frame= were not visible to
  Emacs.
- =e-workspace-visible-window= reused any visible window on the token frame,
  even when the target token named a different workspace than the current one.
- =e-workspace-display-buffer= checked/admitted buffer membership before
  switching to the target workspace, which made Doom/persp admission operate on
  the wrong current workspace.
- =workspace_focus_buffer= preferred the target buffer's existing workspace
  over the active shell workspace, contrary to the model-facing action contract.

The fixes return a cons action for =display-buffer=, only reuse visible windows
when the target token matches the current workspace for that frame, switch
before membership/admission for switch-like policies, and make
=workspace_focus_buffer= target the active shell workspace first.  Regression
coverage now exercises each reviewed failure mode.
