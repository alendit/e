* Review summary

Feature 20 implements the core user-facing goal: Org Canvas buffers show a
context-state indicator, expose a compaction command, and reuse a shared
=e-context-status= computation module instead of duplicating the chat mode-line
logic.  Focused Org Canvas/context-status tests pass, and a live probe confirmed
that an attached Org Canvas buffer renders model/effort/token status and that
=C-c C-m= resolves to =e-org-canvas-compact=.

The review found no evidence that the main happy path is broken.  Two follow-up
notes remain before marking the feature =Done=: missing-session display behavior
does not match the plan's self-check, and compaction-staleness behavior needs a
narrow regression test.

* What looks good

- =e-context-status.el= cleanly owns token-limit lookup, token formatting,
  latest token-usage lookup, compaction-aware token-usage staleness checks,
  context-token estimation, estimate caching, and final text formatting.
- Chat delegates through compatibility helpers such as
  =e-chat--format-token-count=, =e-chat--format-mode-line-status=,
  =e-chat--model-context-token-limit=, and =e-chat--context-token-estimate=.
  The chat prefix and chat customization surface are preserved.
- Org Canvas stores a buffer-local estimate cache, writes the status into
  =mode-name=, restores the prior mode-name state when disabled, and subscribes
  to harness events only for the attached session.
- =e-org-canvas-compact= delegates to =e-chat-session-compact=, so compaction
  policy remains in the shared session/harness action rather than becoming an
  Org Canvas-specific implementation.
- The Org Canvas shell manifest exposes a session-scoped =compact= command and
  the mode keymap binds =C-c C-m= to =e-org-canvas-compact=.
- The shared module does not mutate presentation buffers; shells own rendering,
  mode-line/mode-name state, subscriptions, and cache cells.

* Findings

** TODO Medium: missing-session status does not return the bare prefix promised by the plan

The plan/self-check says missing session state should yield the bare prefix text.
However, =e-context-status-text= currently enters the main formatting path when
both =harness= and =session-id= are non-nil, even if the session id does not
exist.  It then swallows context/option lookup failures with =ignore-errors= and
formats unset model/effort text.

Evidence from this review pass:

#+begin_src emacs-lisp
(e-context-status-text h "missing" :prefix "Org Canvas")
#+end_src

Observed:

#+begin_example
"Org Canvas model unset/effort unset"
#+end_example

Expected per plan/self-check:

#+begin_example
"Org Canvas"
#+end_example

This is not a happy-path blocker, but it can make a detached/deleted session
look like a valid status line with missing options.  Add a test for a missing
session id and either return the bare prefix or explicitly document a different
fallback policy.

** TODO Low: compaction-staleness behavior lacks direct regression coverage

=e-context-status-text= contains compaction-aware logic that ignores token-usage
events predating the latest valid compaction, but neither
=test/e-context-status-test.el= nor =test/e-org-canvas-test.el= has a focused test
for that branch.

Add a narrow test that creates token usage, appends a later compaction, and
asserts that status text falls back to an estimate instead of using stale
pre-compaction token counts.

* Architecture review

The decomposition matches the feature direction well:

- =e-context-status= owns shell-agnostic computation and formatting.
- Chat and Org Canvas own presentation state, subscriptions, cache cells, and
  commands.
- Compaction remains a shared session/harness action reached through the chat
  session capability.

The dependency direction is acceptable: the shared presentation module depends on
stable harness/session reads, not on chat or Org Canvas.  Shells depend inward on
the shared formatter.  The only policy gap found is the missing-session fallback,
which should be resolved inside =e-context-status= rather than papered over in a
shell.

* Design self-check

1. If this work stopped here, E is better off: Org Canvas users now have context
   usage visibility and compaction parity with chat.
2. This moves toward shell-agnostic presentation of harness/session context
   state.
3. Stable decisions: status text is computed from =(harness session-id)= and
   caller-owned cache/config.  Local/reversible decisions: exact mode-name
   rendering and key binding.  Unsettled: missing-session fallback semantics.
4. =e-context-status= owns computation/formatting; each shell owns its own
   render surface and refresh policy.
5. Coupling is reduced by replacing duplicated chat-specific computation with a
   narrow formatter/text API.
6. No dependency points from stable runtime policy toward concrete presentation
   shells.
7. Side effects stay in shells.  The shared module only mutates a caller-owned
   cache cons when supplied.
8. The abstraction is semantically real: multiple shells can use the same
   formatter by supplying a prefix and cache.
9. Chat compatibility helpers/customs are preserved.  No obsolete public path was
   introduced.
10. Duplicated chat computation was removed in favor of the shared module.
11. Expected missing-session behavior needs tightening.  Unexpected harness
    errors are otherwise allowed to surface or are explicitly ignored for status
    fallback.
12. Tests prove shared formatting/estimation and Org Canvas wiring.  Missing
    narrow tests are missing-session fallback and post-compaction stale usage.
