* Implementation Summary

- =e-session.el= maintains tail cells for append-only session lists:
  =:messages=, =:activity-events=, =:branch-summaries=, and =:compactions=.
- =e-session--append-list-item= mutates the previous tail cell in the normal
  initialized case and repairs a missing tail once for legacy or manually
  constructed state.
- Session creation, replay finalization, and list replacement initialize or
  reset the cached tail fields.
- Message-derived metadata is maintained incrementally on append instead of
  calling the full derived-field refresh for every message.
- Public accessors still return copies for append-only list fields.
