* Status
Done

* Note
Adjust =s-i= reference-context insertion so it copies the reference into the
currently visible chat when there is one. Only when no chat is open and visible
should it fall back to the last session.

This is safe to treat as tiny because it should only change target selection for
the existing inline-reference flow. It should not change reference formatting,
session storage, backend behavior, or prompt submission.

* Validation

- visible chat gets the inserted reference
- no visible chat falls back to the last session
- focused =e-chat= tests cover the behavior:
  =rtk eldev test -f test/e-chat-test.el e-chat-test-add-context-to-latest-targets-visible-session e-chat-test-add-context-to-latest-falls-back-to-most-recent-session e-chat-test-add-context-to-latest-deactivates-source-region=
- full chat tests pass with =rtk eldev test -f test/e-chat-test.el=

  session in the running Emacs
