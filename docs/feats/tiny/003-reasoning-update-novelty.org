#+startup: overview

* Status

Done

* Summary

Tighten the =harness-base= model-facing instruction so reasoning updates are
only emitted when they convey specific new information: a decision, tradeoff,
uncertainty, blocker, course correction, or result from inspected context.

The guidance explicitly rejects routine continuation updates, plan restatements,
next-step narration, and repeated reasoning.  When an update is warranted, it
should be one short sentence.
