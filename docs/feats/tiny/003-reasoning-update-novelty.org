#+startup: overview

* Status

Done

* Summary

Balance the =harness-base= model-facing instruction so reasoning updates are
brief but still emitted when they help the user follow meaningful progress or
new context: what the agent learned, a decision, tradeoff, uncertainty, blocker,
course correction, or why a non-obvious next action matters.

The guidance suppresses only boilerplate continuation updates, repeated
plan/status restatements, obvious next-step narration without new context, and
repeated reasoning.  Each emitted update should be one short sentence.
