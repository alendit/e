#+title: Low-Interruption Dev Reload Guidance
#+startup: overview

* Change

External agents now validate =e= changes outside the user's running Emacs by
default.  Full =e-dev-reload= is no longer the default completion step; it is an
explicit user option for when the running Emacs should pick up a change
immediately.

The =e-dev= layer now exposes lightweight reload notification actions.  Internal
=e= agents can mark that a full reload is required and inspect the pending
status without loading source, compiling, running startup hooks, or interrupting
active work.

* Why tiny

This is a narrow developer-workflow adjustment.  It adds a small in-memory
pending-reload marker and two dev-layer actions, but does not implement the
future reload coordinator or scheduled reload machinery.
