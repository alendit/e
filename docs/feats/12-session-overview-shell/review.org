* Implementation Summary

- =e-chat-switch-session= preserves the existing resume behavior by delegating
  to =e-chat-resume=.
- =e-chat-overview= opens a side-window overview buffer listing persisted chat
  sessions in the order returned by =e-harness-session-list=.
- Overview rows show title, short session id, message count, last message time,
  summary, and an unread marker when a loaded session has assistant output newer
  than the presentation read marker.
- Overview read markers are presentation-owned JSON state, stored in the active
  session store directory by default.
- Opening a session from the overview reuses =e-chat-open= and records the latest
  assistant message marker as read.
- The overview subscribes to harness events and refreshes while its buffer is
  live.
- The chat shell manifest advertises =switch-session=, =overview=, and
  =overview-close=.

* Review Notes

Unread detection avoids transcript loads during listing. For sessions that are
already loaded, it tracks the latest assistant message id or timestamp. For
index-only historical sessions, no unread marker is inferred until opening or
another path loads the transcript.
