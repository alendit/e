* Review loop

The first subagent review found one actionable issue:

- duplicate relative =@= file candidates across workspace roots were shown with
  identical labels, making the secondary root impossible to select reliably.

The fix qualifies duplicate labels with their workspace root while leaving
unique labels unchanged, and adds a regression that selects the secondary
duplicate file and verifies the submitted reference content.

Follow-up review on <2026-06-21 Sun> found two related UX robustness issues:

- =@= file completion raised when any configured workspace root was stale or
  missing, even when other workspace roots were still valid.
- =@= and =/= prefix completion used an external picker path instead of keeping
  autocomplete visually inline with the composer.

The fix skips missing workspace roots during file candidate collection and
routes =@= and =/= through an in-buffer inline completion popup.  Regressions
cover stale root handling, =@= inline file selection, and =/= inline prompt
selection without calling =completing-read=.

* UX and performance playthrough

The reviewed user journey is invoking composer prefix file selection when
multiple workspace roots contain the same relative file name.  The fix keeps
unique labels short while qualifying duplicates by workspace root, and the
bounded live probe confirmed the duplicate candidates render as distinct labels.

Follow-up UX review requires =@= and =/= autocomplete to stay inline with the
composer instead of opening a picker in a separate window.  The inline popup is
owned by the composer command, appears at point, supports filtering and keyboard
navigation, and is removed after selection or cancel.  Stale workspace roots are
ignored during =@= candidate collection so one missing directory does not break
autocomplete for the remaining live roots.

The interaction should remain keyboard-friendly because duplicate file labels
are disambiguated before selection rather than relying on hidden metadata, and
because completion stays in the composer surface.  No profiling was required;
the label qualification and stale-root skip are bounded by the candidate set
already presented by autocomplete.
