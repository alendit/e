* Scope Reviewed

Reviewed the implementation against =plan.org= for feature 05, focusing on live
chat progress rendering, provider request boundary replay, and final summary
collapse.

The reviewed changes cover:

- minute/second duration formatting for turn and thought durations;
- structured provider round-trip activity records in the chat presentation;
- live =Thinking...= lines while provider requests are open;
- completed =Thought for <duration>= lines when provider requests finish;
- settled summary blocks for turns with provider boundary activity;
- inline expansion text for settled summaries;
- durable replay of provider boundary events into settled summaries;

* Plan Acceptance Review

** Accepted

- Active provider requests render =Thinking...=.
- Finished provider requests render =Thought for <duration>=.
- Durations use the planned =Nmin Nsec= format.
- Turns with provider boundary events collapse to =Turn took <duration>, <N>
  tool calls.= after the final assistant response.
- Singular tool text is handled for one tool call.
- Summary blocks are navigable and expand to the stored per-line progress
  history.
- Existing final assistant rendering, composer redraw behavior, and block
  navigation tests remain stable.
- Durable replay reconstructs a settled summary when provider boundary events
  are present.

** Remaining Notes

- Expanded summary details are rendered as inline text under the summary block.
  A later polish pass could make each expanded summary line its own navigable
  child block, but the summary expansion preserves the planned visible history.
