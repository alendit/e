#+title: Agents Std Context
#+startup: overview

* Goal

Create an =agents-std-context= layer that contributes standard agent
orientation without requiring the model to ask for it first.

* Design

- The layer is stateless. It contributes one capability with a context provider
  for discovered =AGENTS.md= files and discovered skill resources.
- =AGENTS.md= discovery includes global agent instructions and project
  instructions found by walking up from the layer root.
- Skill discovery reads global =~/.agents/skills/*/SKILL.md= and project
  =.agents/skills/*/SKILL.md= directories, advertises compact references in the
  capability instructions, and exposes full bodies as ordinary read-only
  =e://= resources.
- Default chat harnesses activate this layer through the built-in layer
  registry so AGENTS context participates in normal turn context construction.

* Skill precedence

- Skills are scoped by source. Global skills live under
  =e://agents-std-context/skills/global/<slug>=. Project skills live under
  =e://agents-std-context/skills/project/<slug>=.
- The scope is part of the resource path, so same-slug global and project skills
  are distinct resources and can both be advertised.
- Project skills are discovered from ancestor =.agents/skills= directories.
  When multiple project directories define the same slug, the nearest directory
  to the active project root wins for that project-scope URI.
- Non-duplicate project skills from parent directories remain available.
- Prompt catalog ordering is stable: global skills first, then project skills
  from outer roots to inner roots after duplicate resolution.
