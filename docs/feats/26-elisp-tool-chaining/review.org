** Finding 1: Nested call metadata was not emitted on activity payloads

*** Problem

=e-tools-call= accepted =:metadata= in its options and copied it onto the
nested call plist, but the harness nested executor did not expose that metadata
on the emitted =tool-started= and =tool-finished= payloads.  That missed the
plan requirement that nested-call metadata is merged into visible activity.

*** Change

=e-harness--nested-tool-payload= now appends metadata from the nested tool call
to the event payload.  The nested lifecycle test now passes
=:metadata (:purpose "chain-test")= through =e-tools-call= and asserts both the
started and finished nested payloads expose =:purpose=.

** Finding 2: Elisp eval guidance omitted the required compact example

*** Problem

The =elisp-eval= capability guidance documented the API signatures and decision
rules, but it did not include the compact =search= and =read= example required
by Slice 3.

*** Change

=e-elisp-eval-instructions= now includes a short =e-tools-call!= example that
uses =search= to collect file hits and =read= to fetch each matching resource.
The capability test now checks that the guidance includes both example calls.
