* Architecture self-check

1. If this work package stopped here, the project is better off: representative
   long-running surfaces now share one lifecycle and handle shape.
2. It moves toward the final direction where interactive =e= code starts work
   through policy-declared APIs instead of inline unbounded calls.
3. Stable decisions: every work start returns a handle, policy is explicit, and
   waits are batch/test-only.  Uncertain decisions such as concurrency/coalescing
   enforcement remain local metadata until more call sites need them.
4. =e-work= owns lifecycle and carriers.  Tools/actions/resources/render/task
   code own only their adapters and result shaping.
5. Coupling is reduced by replacing parallel async contracts with a narrow
   consumer-shaped work spec.  Compatibility fallbacks remain for unmigrated
   code.
6. Dependencies still flow from adapters toward the core work contract; core
   policy does not depend on UI, web, resource, or Grimoire modules.
7. Side effects live in carriers or adapters: processes, URL callbacks, timers,
   and queue enqueue stay outside pure result shaping.
8. Added state is handle/lifecycle state with request-lifetime durability only.
   Existing render/task/resource state stays with its current owner.
9. Performance impact is intended positive for interactive paths: starts return
   with handles, and controlled delayed-completion tests prove shape rather than
   stopwatch thresholds.
10. The abstraction is semantically real: carriers all provide start/status/
    cancel/settle behavior without forcing unsupported operations on callers.
11. Compatibility remains explicit.  Old =:handler= and =:start= paths stay only
    as migration fallbacks.
12. Legacy code removable next: broad sync helper entrypoints and ad hoc start
    wrappers after enough call sites move to =:work=.
13. Expected carrier failures settle through =e-work-fail=; unexpected errors
    surface through callbacks or batch await.
14. Core lifecycle tests are independent of the full system, and adapter tests
    prove each representative surface uses the work contract while preserving
    result shape.
