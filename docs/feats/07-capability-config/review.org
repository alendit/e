  exact against skill name or resource path, exclude wins, and filtering applies
  to advertised skill catalog entries and readable =e://= skill resources.
- AGENTS.md discovery remains separate from skill filtering.
- =docs/architecture.md= documents that configuration is capability-owned and
  layers remain stateless presets.

* Deviations And Risks

- Project skill filtering resolves config for the active session context root at
  provider/resource time, because existing =agents-std-context= behavior lets a
  session project root override the layer construction root. The constructed
  capability still stores the effective config for its construction root. This
  preserves current session-root behavior while keeping config validation at the
  capability boundary.
- Include/exclude matching is intentionally exact only. Pattern matching remains
  later work, matching the plan.
