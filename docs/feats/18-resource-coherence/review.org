#+title: Resource Coherence Review
#+startup: overview

* Status

Ready for review.

* Summary

Implemented resource coherence as a generic linked-resource model, with the
local file/file-backed-buffer case as the first concrete provider.

Generic model added in =e-resource-coherence=:

- coherence registries that dispatch to providers by URI scheme;
- provider objects that return a coherence group for a URI;
- arbitrary-size coherence groups containing any number of linked resource views;
- view metadata for canonical URI, kind, role, status, modified/dirty state,
  liveness, visibility, selected-window state, priority, and adapter metadata;
- generic aggregate status, dirty-view conflict detection, preferred-view
  selection, and view filtering helpers.

The existing local file/buffer behavior now uses that generic model:

- =file://= reads prefer a preferred live view from a coherence group;
- =file://= writes/edits ask generic dirty-view policy before mutating;
- modified visiting buffers remain explicit coherence conflicts;
- unmodified visiting buffers are updated through the live buffer and saved;
- other unmodified linked buffers are reverted after save instead of left stale;
- =resource_sync_status= reports the generic group and still includes the
  compatibility-oriented file/buffer fields that existing tests and agents use.

* Notes

- Canonical path comparison uses =file-truename= so macOS =/tmp= and
  =/private/tmp= aliases resolve to the same live buffer.
- The sync status tool is registered by both file inspection and file mutation
  capabilities, so it is available even with read-only file access.
- The generic model is deliberately provider-shaped rather than a persisted
  global graph. It can represent any number of linked resource views for a
  canonical entity while keeping concrete discovery/sync mechanics in adapters.
