** Finding 1: =e-debug= created a fresh tab on every invocation

*** Problem

The tab display strategy called =tab-bar-new-tab= before showing the debug
buffer, so repeated =e-debug= invocations kept creating extra tabs instead of
returning to a stable debug workspace.

*** Change

=e-debug-tab-name= now names the debug tab.  =e-debug--show-buffer= selects the
existing tab with that name when available, otherwise it creates and renames one
debug tab before showing the standing session buffer.

** Finding 2: =e-debug-here= lost active chat identity without a recent failure

*** Problem

When =e-debug-here= was invoked in an =e-chat-mode= buffer with no recent
failure, the debug submission had no =:inspection-session-id=.  The debug agent
could see focused source context but not which chat session was being inspected.

*** Change

=e-debug--inspection-target= now captures the active chat harness and session id
from =e-chat-mode=.  =e-debug--capture= carries that session id into submission
metadata and adds an =inspected-session= reference alongside source and failure
references.

* Floating popup addendum review loop

** Finding 3: notification subscription did not follow a changed debug session id

*** Problem

The addendum introduced a session-filtered notification subscription, but the
first implementation keyed reuse only by harness.
If the standing debug session id changed inside the same harness, the old
subscription would remain attached to the previous session and background
completion notifications could be missed for the new debug session.

*** Change

=e-debug--notification-session-id= now records the subscribed session id.
=e-debug--ensure-notification-subscription= re-subscribes when either the
harness or session id changes.

*** Evidence

#+end_src

#+end_example

** Finding 4: stale popup state could suppress background completion messages

*** Problem

The addendum hides notifications while the floating popup is visible, but the
first visibility check looked only at the popup buffer and minor mode.
If the child frame were deleted outside the normal dismiss path while the minor
mode remained active, =e-debug= could treat the popup as visible and suppress the
=*e-debug*:= completion message.

*** Change

=e-debug--popup-visible-p= now also requires the recorded popup frame to be live
when one exists.
The normal =C-g= dismiss path still clears the buffer and frame state.
