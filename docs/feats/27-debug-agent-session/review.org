** Finding 1: =e-debug= created a fresh tab on every invocation

*** Problem

The tab display strategy called =tab-bar-new-tab= before showing the debug
buffer, so repeated =e-debug= invocations kept creating extra tabs instead of
returning to a stable debug workspace.

*** Change

=e-debug-tab-name= now names the debug tab.  =e-debug--show-buffer= selects the
existing tab with that name when available, otherwise it creates and renames one
debug tab before showing the standing session buffer.

** Finding 2: =e-debug-here= lost active chat identity without a recent failure

*** Problem

When =e-debug-here= was invoked in an =e-chat-mode= buffer with no recent
failure, the debug submission had no =:inspection-session-id=.  The debug agent
could see focused source context but not which chat session was being inspected.

*** Change

=e-debug--inspection-target= now captures the active chat harness and session id
from =e-chat-mode=.  =e-debug--capture= carries that session id into submission
metadata and adds an =inspected-session= reference alongside source and failure
references.
