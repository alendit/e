* Final review

Initial subagent review found five implementation issues:

- adapter calls did not match current =agent-shell--start=,
  =agent-shell-insert=, and =agent-shell-subscribe-to= keyword APIs;
- adoption accepted spoofed buffer-local metadata from non-Agent-Shell buffers;
- dead buffers could fall through =e-agent-shell-status=;
- =:list-work= and =:work-status= returned stale registry status;
- the capability existed but was not registered as a default layer.

Those findings were addressed with failing regression tests first, then code
changes in the adapter, work registry, fleet capability, and default layer
registration.

Second subagent review found four additional contract gaps:

- prompt insertion did not pass =:submit t=, so handoff/follow-up could insert
  text without submitting it to Agent Shell;
- Agent Shell alist events with nested =:data= were not normalized by the work
  registry;
- live status refresh could overwrite terminal event-derived states like
  =finished= with coarser adapter status like =ready=;
- adoption still accepted a plain buffer that spoofed =agent-shell--state=.

Those findings were addressed with failing regression tests first, then code
changes to submit prompts, require Agent Shell mode plus matching state during
adoption, read nested alist event payloads, and preserve terminal work states
during refresh.
