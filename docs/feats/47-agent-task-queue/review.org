* Suggested review use cases

Derived from =plan.org= acceptance, plus surface-driven additions:

- Enqueue returns a =queued= record immediately; dispatch may start it before
  return.
- With cap 2, a third enqueue waits until one running task settles.
- =queued= -> =running= -> =done= over a real harness turn; =read-task= returns
  outputs.
- Failing turn lands =failed= with =:error= and does not block the dispatcher.
- Cancel of a queued task: =cancelled=, never runs; cancel of a running task:
  interrupts and a late settle does not resurrect it.
- Per-task =harness-instance-id= wins; omitted uses the queue default;
  unregistered id fails just that task without stalling dispatch.
- List shell: opens newest-first, refreshes on queue change, row cancel and
  show-outputs, workspace-aware display; empty-queue state; repeated use.
