** Findings

*** DONE Goal belongs in harness-base

The capability models a general controller loop for agent work.
It does not depend on chat, Org Canvas, filesystem tools, web tools, or project-local code.

*** DONE Subagent execution remains separate

The capability returns executor-neutral next-action prompts.
A separate subagent capability can consume those prompts later and own worker spawning, split display, callbacks, and transcript reading.

*** DONE Achievement is deterministic

The =:assess-goal= action computes achievement from steps, criteria, and blockers.
Stopped or blocked goals are not treated as achieved.

*** DONE Persistence is out of scope for the first version

The current registry is in-memory.
That is enough for active controller flows.
If goals need to survive Emacs restart, add persistence in a separate slice.
