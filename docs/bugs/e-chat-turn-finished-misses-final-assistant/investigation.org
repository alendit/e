#+title: e-chat turn-finished misses final assistant investigation

* Evidence

- Computer-use inspection showed the =Debug Agent= popup still displaying the
  older reasoning/progress transcript rather than the final assistant answer.
- The durable session
  =~/.emacs.d/.local/cache/e/sessions/sessions/20260621T193605-da08c5bf25c1.jsonl=
  had a final assistant message and =turn-finished= for turn
  =01KVNXZ1AHZZCVACSDCACJDC71= at =2026-06-21T20:47:41Z=.
- A live scalar probe before Emacs became unresponsive showed the harness was
  idle: =:active-turn nil=, =:progress-turn nil=, =:progress-timer-live nil=,
  and =:status "idle"=.
- The visible chat buffer's turn registry had activity rendered for
  =01KVNXZ1AHZZCVACSDCACJDC71=, but =:final-rendered nil=.  The buffer tail did
  not include the durable final assistant text.

* Root Cause

The chat renderer relied on receiving and rendering the final assistant
=message-added= event.  If that presentation event was missed, delayed, or
lost across the reload/rebind churn, the later =turn-finished= event settled
status and composer state without reconciling the transcript against durable
session messages.

The provider/session state was correct, but presentation did not catch up from
the durable source of truth, so the UI could keep showing stale activity and
look hung even though no turn was active.

* Fix

- Added =e-chat--final-assistant-message= to find the durable assistant message
  for a turn in the attached session.
- Added =e-chat--render-missed-final-assistant=.
- Updated the =turn-finished= renderer to render that durable final assistant
  when the turn record has not already marked =:final-rendered=.
