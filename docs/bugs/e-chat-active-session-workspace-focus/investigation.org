#+title: e-chat active session workspace focus investigation
#+startup: overview

* Investigation

=e-chat-active-sessions= delegates candidate selection to
=e-chat--active-session-open=, which calls =e-chat-open-session=.
=e-chat-open-session= calls =e-chat-open= and then =e-chat--pop-to-buffer= when
display is requested.

Two root causes made this path select the wrong workspace:

- =e-chat--active-session-preview= renders the picker preview in =e-chat-mode=
  and sets =e-chat-harness= plus =e-chat-session-id=.  =e-chat--find-session-buffer=
  accepted any matching =e-chat-mode= buffer, so the visible preview could be
  mistaken for the canonical session shell.
- Reattaching an existing chat buffer ran =e-chat-mode= before checking the
  buffer's workspace affinity.  Since =e-buffer--workspace= is buffer-local,
  that mode reset erased the old affinity and =e-chat--attach-buffer= rebound
  the shell to the current workspace.

* Fix

=e-workspace-find-buffer= is now the shared helper for presentation shells that
need to find an existing canonical buffer before creating or opening another
one.  It accepts a predicate and centralizes preference for visible buffers and
buffers already bound to the target workspace.

Chat preview buffers now carry a buffer-local =e-chat--preview-buffer= marker.
=e-chat--matching-session-buffer-p= excludes those transient previews from
canonical session lookup, and =e-chat--find-session-buffer= delegates matching
preference to =e-workspace-find-buffer=.

=e-chat--attach-buffer= now captures the existing workspace token before
reinitializing =e-chat-mode= and restores it unless
=e-workspace-rebind-shell-on-open= explicitly requests rebinding.

=e-org-canvas-session-buffer= also delegates live canonical canvas lookup to
=e-workspace-find-buffer= before it falls back to buffer-name, file, or
attachment-based recovery.
