#+title: e-chat overview renders raw prompt context investigation
#+startup: overview

* Investigation

Live Emacs inspection showed that =*e-chat-overview*= was correctly in
=e-chat-overview-mode=, with no local =e-chat-mode= session attachment. The
buffer was not an accidental chat buffer reuse.

The root cause was =e-chat-overview--insert-session-row= inserting raw
=:title= and =:summary= fields from =e-harness-session-list=. Those fields can
come from the first user prompt, which may include multiline context/reference
payloads. The overview renderer therefore expanded one logical row into prompt
context and reference blocks.

* Fix

The overview renderer now strips inline reference tags, ignores raw reference
listing lines, compacts row title and summary text to the first meaningful line
before insertion, and caps the displayed length. It also avoids printing the
summary when it is identical to the compacted title.

* Verification

Added =e-chat-test-overview-compacts-multiline-session-summary=, which failed
before the fix because the rendered overview contained =<reference= and
=References:= text.
