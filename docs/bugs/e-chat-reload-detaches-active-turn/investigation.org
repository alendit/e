#+title: e-chat reload detaches active turn investigation

* Evidence

- The current debug session file was
  =~/.emacs.d/.local/cache/e/sessions/sessions/20260621T193605-da08c5bf25c1.jsonl=.
- The older turn =01KVNXDQYDP7M2SSYEFCKVS0KQ= continued appending provider
  requests, tool calls, tool results, and reasoning deltas after the user had
  nudged the session.
- The nudge prompt produced later turn ids, including
  =01KVNXWCDEC4TD19QS4159B961= and =01KVNXZ1AHZZCVACSDCACJDC71=.
- The durable log showed interleaved activity from the old turn and newer
  turns in the same session.  One newer-turn message was appended under a
  parent id belonging to the old turn's provider request chain, which is
  consistent with two harness/session writers touching the same durable
  session.
- A live scalar probe initially showed no stale progress timer in the visible
  =*e-chat:Debug Agent*= buffer.  Later the current active turn rendered
  normally, so the first stale-timer fix was not the whole failure.

* Root Cause

=e-dev-reload= reloads =e-chat.el= and then reruns startup hooks.  Chat startup
calls =e-chat-reload-buffers=.

Before this fix, =e-chat-reload-buffers= always chose the currently configured
default or instance harness for each live chat buffer when that harness could
be created.  That is correct for idle buffers, but unsafe for a buffer whose
current harness still has a running active turn for the same session.

In that active-turn case, reattaching the buffer to a fresh/current default
harness detaches the presentation from the harness object whose async provider
callbacks still own the in-flight turn.  The old callbacks can continue
writing durable activity, while the visible buffer now consults a different
harness state and can accept new input as if there were no active turn.  That
creates the observed split-brain state.

* Fix

- Added =e-chat--harness-session-active-turn-p=.
- Updated =e-chat-reload-buffers= to preserve the buffer's current harness
  when that harness has a running active turn for the buffer's session.
- Kept the existing idle-buffer behavior: inactive buffers still refresh to
  the current default or instance harness on reload.
